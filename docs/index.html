How It Works

This pipeline transforms raw transaction data into investigation-ready fraud case narratives through a multi-stage AI pipeline:

  • Data Ingestion: Card transaction data undergoes comprehensive feature engineering, extracting temporal patterns, merchant analytics, and customer behavior profiles.
  • Anomaly Detection: XGBoost classifier trained on historical fraud patterns identifies suspicious transactions with high precision.
  • Explainability: SHAP (SHapley Additive exPlanations) generates local and global interpretability, quantifying each feature's contribution to the fraud score.
  • Silver Labeling: SHAP outputs are converted into structured natural language descriptions that serve as training signals for the language model.
  • Narrative Generation: Fine-tuned Flan-T5 (Small Language Model) generates human-readable fraud case summaries, complete with risk assessments and investigative recommendations.
  • Output Artifacts: Generated narratives, SHAP visualizations, and case metadata are saved for fraud analyst review and audit trails.

How to Reproduce

To run the full pipeline on your own machine:

Prerequisites

  • Python 3.8+
  • Required packages: transformers, torch, xgboost, shap, pandas, scikit-learn

Quick Start

# Clone the repository git clone https://github.com/pranshu-dhingra/AI-Driven-Fraud-Case-Summarizer.git cd AI-Driven-Fraud-Case-Summarizer # Install dependencies pip install -r requirements.txt # Run the complete pipeline bash run_pipeline.sh

Individual Pipeline Steps

You can also run each module independently:

python -m src.data_preprocessing python -m src.feature_engineering python -m src.customer_profile_features python -m src.velocity_features python -m src.anomaly_labeling python -m src.xgb_interpret python -m src.shap_enrichment python -m src.slm_train python -m src.inference

Generated narratives and SHAP visualizations will be saved in the outputs/ directory.

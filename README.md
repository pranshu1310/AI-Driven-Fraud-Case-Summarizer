# Generative AI â€“ SLM-based Fraud Case Narrative Generator

This project builds an end-to-end pipeline that:

1. Ingests card transactions data and applies feature engineering.
2. Trains an XGBoost anomaly classifier and explains predictions with SHAP.
3. Converts SHAP outputs into natural-language "silver labels".
4. Fine-tunes a small language model (Flan-T5) to generate fraud case narratives for each transaction.

### Key folders

- `data/raw/` â€“ raw CSV (e.g., `transactions_raw.csv`).
- `data/processed/` â€“ feature-engineered and SHAP-enriched datasets.
- `models/` â€“ saved XGBoost model, SHAP global importance, and SLM model.
- `outputs/` â€“ generated narratives for manual review.

- `data_preprocessing.py` â€“ load and clean raw data.
- `feature_engineering.py` â€“ build structured features for modeling.

- `xgb_interpret.py` â€“ train XGBoost, compute SHAP global and local explanations.

- `slm_train.py` â€“ fine-tune a sequence-to-sequence model (Flan-T5).
